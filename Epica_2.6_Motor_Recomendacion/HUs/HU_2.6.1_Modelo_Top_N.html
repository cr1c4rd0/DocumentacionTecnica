Historia de Usuario · HU 2.6.1

Modelo de recomendación de servicios

Enunciado

COMO

usuario del portal CCB

QUIERO

recibir una lista Top-N de servicios con scores explicables según mi perfil completo

PARA

descubrir los servicios más relevantes para mi situación empresarial

Criterios de Aceptación
  • Motor genera lista Top-N (N configurable, defecto 5)
  • Scores basados en: perfil, historial, segmento e intereses
  • Cada recomendación incluye razón legible en lenguaje natural
  • Filtros de elegibilidad aplicados antes de retornar resultados
  • Latencia p99 ≤ 1 segundo
Notas Técnicas
  • Modelo híbrido (colaborativo + contenido)
  • Feature store con features precalculadas
  • API asíncrona con fallback sincrónico
  • Versionamiento semántico del modelo
