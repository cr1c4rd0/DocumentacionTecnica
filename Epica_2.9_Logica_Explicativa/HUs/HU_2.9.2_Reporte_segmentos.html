Reporte de segmentos

Épica 2.9 · Lógica Explicativa · Dashboard de métricas segmentadas del recomendador

Enunciado

COMO

usuario de negocio (líder/analista responsable del Recomendador de Servicios CCB).

QUIERO

acceder a un reporte / dashboard con métricas segmentadas del recomendador: volumen de recomendaciones, tasa de aceptación, conversión y distribución de servicios por segmento de usuario.

PARA

entender el comportamiento del motor por grupos de usuarios, identificar oportunidades de mejora, detectar brechas y tomar decisiones estratégicas basadas en datos.

1. Criterios de Aceptación Reporte de Segmentos
  1. Disponibilidad de reporte segmentado
    • Existe un dashboard/reporte accesible desde el Panel de Administración del Recomendador, sección "Reportes de Segmentos".
    • El reporte presenta métricas desglosadas por segmento de usuario (MiPyme, Empresa Grande, Ciudadano, Emprendedor, etc.).
    • KPIs globales visibles al inicio para panorama rápido: total de recomendaciones generadas, tasa de aceptación global, tasa de conversión global, top 5 servicios más recomendados.
  2. Trazabilidad de métricas y definiciones
    • Cada métrica tiene una definición documentada accesible con tooltip o enlace "¿Cómo se calcula?":
      • Volumen: número de recomendaciones generadas por segmento en el período.
      • Tasa de aceptación: % de recomendaciones en las que el usuario hizo clic en "Ver detalle" o aplicó un CTA.
      • Tasa de conversión: % de recomendaciones que resultaron en inscripción/contratación confirmada.
      • Distribución de servicios: breakdown de qué servicios se recomiendan más por segmento.
    • Las métricas son trazables hasta la fuente: pipeline de agregación batch con job_id y run_timestamp.
  3. Rendimiento y actualización
    • Los datos del reporte se actualizan con frecuencia configurable (defecto: diaria, pipeline nocturno).
    • El dashboard indica claramente la fecha y hora de la última actualización.
    • Los resultados del reporte se cachean por 24 horas con posibilidad de invalidación manual (botón "Actualizar ahora" para admin).
    • El tiempo de carga del dashboard (con caché caliente) es ≤ 3 s.
  4. Manejo de datos incompletos (fallback analítico)
    • Si un segmento no tiene datos suficientes para el período seleccionado, se muestra una alerta contextual: "Datos insuficientes para este segmento en el período. Se requieren al menos N registros para calcular métricas estables."
    • El reporte sigue funcionando para los segmentos con datos suficientes.
    • Opción de ver datos del período anterior cuando el actual no está disponible (fallback analítico).
  5. Exportación / consumo por negocio
    • Exportación del reporte completo en CSV y Excel (.xlsx).
    • Exportación de gráficos individuales en PNG.
    • Opción de configurar un envío periódico por correo (frecuencia: semanal/mensual) a una lista de destinatarios definida por admin.
  6. Acceso y configuración de filtros
    • El usuario accede desde el sidebar de navegación del Panel Admin → sección "Reportes".
    • Panel de filtros con opciones:
      • Fecha: rango personalizado, últimos 7/30/90 días, mes/trimestre actual.
      • Segmentación: tipo de usuario (MiPyme, Empresa, Ciudadano, etc.).
      • Servicio: filtrar por categoría o servicio específico.
      • Canal: web, app, chat, presencial.
    • Filtros combinables con actualización del dashboard al aplicar.
  7. KPIs globales (panorama rápido)
    • Sección superior del dashboard con tarjetas de KPI:
      • Total recomendaciones generadas en el período.
      • Tasa de aceptación global (%).
      • Tasa de conversión global (%).
      • Top 3 servicios más recomendados.
      • Segmento con mayor tasa de conversión.
    • Comparación con el período anterior: delta absoluto y relativo con indicador de tendencia (flecha arriba/abajo).
2. Tecnología Stack Propuesto
Pipeline de Agregación
  • Job batch nocturno (cron / Airflow / Databricks Workflow) que agrega métricas por segmento.
  • Fuentes: tabla de recomendaciones, tabla de eventos de interacción, tabla de conversiones (SIREP).
  • Resultado almacenado en Data Warehouse (tabla de hechos por segmento/período).
  • Particionado por fecha y segmento para consultas eficientes.
API de Reportes
  • GET /reports/segments?from=&to=&segment=&canal=&service=
  • Respuesta: KPIs globales + breakdown por segmento + series temporales.
  • Caché de 24 h con invalidación manual vía POST /reports/segments/invalidate.
  • Latencia objetivo (caché caliente): ≤ 1 s.
Frontend del Dashboard
  • Biblioteca de charts: Chart.js (barras, líneas, dona) o D3.js para visualizaciones avanzadas.
  • Tarjetas de KPI con indicadores de tendencia (delta vs. período anterior).
  • Tabla de distribución de servicios por segmento (ordenable y filtrable en cliente).
  • Botones de exportación CSV / Excel / PNG por sección.
Gobernanza y Privacidad
  • Solo se exponen métricas agregadas; nunca datos a nivel de usuario individual en el reporte.
  • Los segmentos con menos de N usuarios se enmascaran (k-anonymity mínima).
  • El acceso al reporte queda registrado en log de auditoría.
  • Datos seudonimizados en el pipeline; PII no transita por la capa de reportes.
3. Flujo Técnico y de Datos Secuencia Dashboard
ACTORES

Usuario / Actor (Líder/Analista) · Motor / API · CRM / Dynamics · Frontend / UI · Formulario

DISPARADOR

El analista necesita revisar el desempeño del motor de recomendación por segmento para un período dado y tomar decisiones estratégicas.

RESULTADO

Dashboard con KPIs globales, métricas por segmento, distribución de servicios, comparación de períodos y opción de exportación.

Pasos del Flujo (según Diagramas)
  1. Disponibilidad de reporte segmentado: el Usuario / Actor solicita el reporte al Motor / API; el API verifica disponibilidad de datos agregados en el DW.
  2. Trazabilidad de métricas y definiciones: el Motor / API consulta el CRM / Dynamics para enriquecer la definición de segmentos; las métricas se calculan con su definición documentada.
  3. Rendimiento y actualización: el Frontend / UI recibe los datos del pipeline nocturno y los renderiza en el Formulario (dashboard); muestra la fecha de última actualización.
  4. Manejo de datos incompletos (fallback analítico): el Formulario detecta segmentos con datos insuficientes y muestra alertas contextuales; el Motor retorna datos del período anterior si aplica.
  5. Exportación / consumo por negocio: el Motor / API genera el archivo CSV/Excel a petición del Usuario; el Frontend lo descarga directamente.
  6. El usuario accede desde el sidebar de navegación: el Motor / API carga la configuración del reporte; el Frontend renderiza el panel con los filtros disponibles.
  7. Configura filtros: el Usuario selecciona fecha, segmentación, servicio, canal; el Frontend actualiza la vista con los datos filtrados desde el DW.
  8. Revisa KPIs globales: el Usuario obtiene el panorama rápido de métricas clave; puede explorar el detalle por segmento desde las tarjetas de KPI.

4. Diseño — Dashboard de Reportes
Sección KPIs Globales
  • 4–5 tarjetas de KPI en fila superior con valor, delta y tendencia.
  • Colores semánticos: verde (mejora), rojo (baja), gris (sin cambio).
  • Tooltip con definición de cada KPI al pasar el cursor.
  • Actualización: fecha/hora de la última ejecución del pipeline.
Sección Breakdown por Segmento
  • Gráfico de barras agrupadas: tasa de aceptación y conversión por segmento.
  • Tabla de distribución de servicios por segmento (ordenable).
  • Comparación side-by-side entre dos segmentos seleccionados.
  • Opción de ver evolución temporal (gráfico de líneas) por segmento.
5. Casos de Prueba Mínimos
# Escenario Resultado Esperado
1 Carga del dashboard El dashboard se carga en ≤ 3 s con caché caliente; muestra KPIs globales y breakdown por segmento.
2 Filtro por segmento y fecha Al filtrar por "MiPyme" y "últimos 30 días", el dashboard muestra solo métricas de ese segmento y período.
3 Exportar en CSV Se descarga un CSV con todas las métricas por segmento del período seleccionado, con encabezados y definiciones.
4 Datos insuficientes en segmento El segmento con pocos datos muestra alerta contextual; el resto del dashboard sigue funcionando.
5 Invalidación manual de caché Al hacer clic en "Actualizar ahora", el pipeline se re-ejecuta y los datos se refrescan; el dashboard muestra la nueva fecha de actualización.
6 Comparación entre períodos Las tarjetas de KPI muestran el delta correcto vs. el período anterior con indicador de tendencia.
7 Sin PII en reportes Ningún dato individual aparece en el reporte; todos los valores son agregados por segmento.
8 Segmento con k-anonymity insuficiente Segmento con menos de N usuarios se enmascara; no se muestra la métrica sino un aviso de privacidad.
6. Glosario
Tasa de aceptación
% de recomendaciones en las que el usuario realizó una acción afirmativa (clic en CTA, "Ver detalle", inscripción iniciada).
Tasa de conversión
% de recomendaciones que resultaron en una inscripción o contratación confirmada (validado en SIREP).
Segmento
Grupo de usuarios con características comunes (tipo de empresa, etapa, sector) usado para segmentar las métricas del motor.
Pipeline batch
Proceso de agregación nocturno que calcula y almacena las métricas segmentadas en el Data Warehouse.
k-anonymity
Propiedad de privacidad que garantiza que cada registro en el reporte es indistinguible de al menos k-1 otros registros del mismo segmento.
Fallback analítico
Estrategia que muestra datos del período anterior cuando el período actual no tiene suficientes registros para métricas estables.
Delta
Diferencia (absoluta y relativa) entre el valor de una métrica en el período actual vs. el período anterior, usada para indicar tendencia.
Nota de alcance: El reporte de segmentos opera sobre datos agregados del pipeline batch nocturno; no expone datos individuales de usuario. La frecuencia de actualización (defecto: diaria) y el umbral de k-anonymity son configurables por admin. La exportación en CSV/Excel y el envío periódico por correo son funcionalidades de distribución del reporte a stakeholders de negocio.
