Historia de Usuario · HU 2.16.1

Captura de feedback en recomendador

Enunciado

COMO

usuario del portal que recibe recomendaciones

QUIERO

indicar si una recomendación es útil o no con un gesto simple y opcional

PARA

ayudar al sistema a mejorar y recibir recomendaciones cada vez más relevantes

Criterios de Aceptación
  • Botones thumbs up / thumbs down visibles pero discretos en cada tarjeta
  • Comentario opcional de texto libre (máximo 280 caracteres)
  • Registro del feedback en ≤ 500ms sin bloquear la UI
  • Confirmación visual inmediata al enviar feedback
  • Feedback neutral es válido (sin forzar interacción)
  • Datos de feedback anonimizables según consentimiento del usuario
Notas Técnicas
  • API POST /feedback con payload mínimo
  • Fire-and-forget desde el cliente con retry silencioso
  • Almacenamiento en tabla de feedback con recommendationId
  • Agregación diaria para alimentar pipeline de reentrenamiento
