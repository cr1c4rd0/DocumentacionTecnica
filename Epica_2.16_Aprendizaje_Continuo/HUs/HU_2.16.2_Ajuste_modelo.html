Historia de Usuario · HU 2.16.2

Ajuste de modelo por feedback

Enunciado

COMO

Data Scientist responsable del modelo de recomendación

QUIERO

usar el feedback e interacciones recolectados para reentrenar y mejorar el modelo periódicamente

PARA

que el motor evolucione con el comportamiento real de los usuarios y mejore sus métricas de negocio

Criterios de Aceptación
  • Pipeline de reentrenamiento consume feedback + interacciones de las últimas N semanas
  • Validación de fairness y sesgo antes de desplegar nueva versión
  • Comparación A/B entre versión actual y nueva candidata
  • Rollback automático si métricas caen más del umbral configurado
  • Notificación al equipo cuando el modelo se actualiza en producción
  • Incremento mínimo de precisión del 2% para aprobar nueva versión
Notas Técnicas
  • MLflow o similar para tracking de experimentos
  • Pipeline orquestado con Airflow
  • Validación de sesgo con Fairlearn o AIF360
  • Blue-green deployment para cambio de versión
  • Alertas en Grafana/Datadog sobre degradación de métricas
